# StreamNotify

- YouTube/ニコニコ動画/Twitch の放送・新着を検知し、  
Bluesky へリアルタイムまたは予約でお知らせ投稿する Python 製 Bot です。
- ニコニコは動画投稿、YouTube は放送と動画投稿の通知に対応します。
- **v4系**が現在のメインバージョンです。GUI・ローカル Webhook（FastAPI）・  
バックグラウンドワーカーを組み合わせた構成で、v3 で分かれていた拡張機能はコアに統合されています。

---

## 主な特徴 (v4)
### 基本機能
- **YouTube**: RSS ポーリングと、センター経由 WebSub（`YOUTUBE_FEED_MODE` で切替）の統合パイプライン。<br>ライブ・プレミア・アーカイブ・予約枠に応じたテンプレート
- **ニコニコ動画**: RSS 監視と Bluesky 投稿
- **Twitch**: EventSub（開始・終了・Raid 等）と OAuth 認証（`websub` モード利用時）
- **モジュール分割 GUI**（`v4/gui/`）: フィルタ・コンテキストメニュー・テーマ（ライト/ダーク/システム）、テンプレートのプレビュー付き編集
- **統合設定画面**: 主要な環境変数を GUI から編集（`settings.env`）。カテゴリ別ログレベルなど

### 投稿関連
- **Bluesky** への自動・手動・予約投稿、Jinja2 テンプレート、画像添付・リンクカード
- **一括操作**: 複数選択での削除・スケジュール予約など

### アーキテクチャ・保守
- **ローカル Webhook サーバー**（既定 `127.0.0.1:20000`）でプッシュ通知を受信。トンネル通信の前提を撤廃
- **Pydantic v2** ベースの設定、SQLite、バックアップ・復元（ZIP）
- **v3 のプラグイン機構は廃止**し、同等機能は標準内蔵

> **Bluesky OAuth**: 実装・設計は進行していますが、**v4.0.0 時点では利用できません**。  
> 当面は **アプリパスワード**での Bluesky 連携を利用してください。

## v3 からの違い（概要）
- プラグインの手動配置は不要（機能はコア統合）。
- WebSub 利用時は **ローカル Webhook** と `WEBSUB_CALLBACK_BASE_URL` / `HOST`・`PORT` の整合が重要。
- 詳細はリポジトリ内 `v4/docs/Guides/v3_to_v4_User_Guide.md` および Wiki を参照してください。

## 起動方法 (v4)
### Windows で迷わない起動（おすすめ）
#### v4から、簡単にインストール、セットアップ、修復、起動ができるランチャーを同梱しています。

1. Release ページから最新の安定版をダウンロードし、ZIP を解凍します。
2. start_launcher.bat をダブルクリックして「セットアップ/修復」をクリックします。
3. セットアップ(起動準備)が完了したらランチャー画面に表示されますので、続けて起動をクリックします。
4. 2回目からは起動をクリックするだけで起動します。

### 手動での起動方法
1. Release ページから最新の安定版をダウンロードし、ZIP を解凍します。
2. **`v4` ディレクトリ**をカレントにし、依存関係を入れます。
   ```bash
   cd v4
   pip install -r requirements.txt
   ```
3. **初回のみ**セットアップを実行します（テンプレート展開・`settings.env` の生成など）。
   ```bash
   python setup_v4.py
   ```
4. `settings.env` に各サービスの ID・キーを設定したうえで起動します。
   ```bash
   python main_v4.py
   ```
   GUI とローカル Webhook サーバーが同時に立ち上がります。

### v3 をお使いの場合
```bash
pip install -r v3/requirements.txt
python v3/main_v3.py
```
（v3 はメンテナンスフェーズです。新規導入は v4 を推奨します。）

## 運用上の注意とセキュリティ
- **マニュアルや技術ドキュメントは添付されていません**:  
ユーザーガイド、GUI操作マニュアル、トラブルシューティング、技術ドキュメントなどは、  
直接Releaseには含んでいません。これらの内容を確認するには[Wiki](https://github.com/mayu0326/StreamNotify/wiki)をご確認ください。
- **APIキーの秘匿**:   
セキュリティのため、**APIキーやパスワード**は絶対に公開リポジトリに含めないでください。  
また、それらの内容は **設定ファイル(`settings.env`)** や、**バックアップファイル（`backup/`以下）**、  
**設定ファイルのバックアップ（`settings.env.backup`）** に含まれています。
- **リソース監査**:  
ログファイル（`logs/`以下）には重要な履歴が記録されます。  
運用時はアクセス権限に注意してください。
- **商用利用**:  
本アプリは個人運用・検証を想定しています。商用利用は自己責任でお願いします。
