# クイックスタート (はじめての準備)

StreamNotify を使い始めるための準備手順を分かりやすく解説します。  
> この文書は現在の最新バージョンであるv3.2.0を基準としています。  
> そのため既存バージョンを利用中の場合、一部の機能や説明が異なる場合があります。

---

## 💻 1. 動作環境の準備
StreamNotify を動かすために必要な環境は以下の通りです。

- **OS**: Windows 10/11, macOS, Linux (Windows 11 または Ubuntu 22.04 推奨)
- **Python**: 3.10 以上 (3.12 以上を強く推奨)
- **ネットワーク**: 常時インターネットに接続されていること
- **必要なアカウント情報等**:
  - 監視したい YouTube チャンネルの ID (`UC` から始まる 24文字)
  - 監視したい ニコニコアカウントのID (ニコニコのユーザー向けページにに記載の `数字ID` です)
  - Bluesky のアカウント情報（ユーザーハンドル名 と アプリパスワード）  
  ※アプリパスワードは、Bluesky の設定画面から作成できる専用のパスワードです。

---

## 🛠️ 2. インストール手順

### ステップ 1: アプリ本体のダウンロード
- 簡単な方法  
トップページにある [ `Releases` ] か、トップページ下段にある [ `ダウンロード` ] をクリックして、  
ご希望のバージョンをダウンロードします。

- Git を使ってダウンロードする方法  
GitHub のリポジトリを `git clone` するか、ZIP ファイルをダウンロードして解凍し、  
その中の `v3` フォルダを開きます。

> [!TIP]
> 以降の作業はすべてこの `v3` フォルダ内で行います。

### ステップ 2: 仮想環境の作成 (おすすめ)
他の Python アプリやシステムとの競合を防ぐため、仮想環境での実行を強く推奨します。  
ターミナル（コマンドプロンプトなど）で以下のコマンドを実行します。

```bash
# 仮想環境の作成
python -m venv .venv

# 有効化 (Windows の場合)
.venv\Scripts\activate

# 有効化 (macOS/Linux の場合)
source .venv/bin/activate
```

### ステップ 3: 必要なパッケージのインストール
仮想環境を有効にした状態で、以下のコマンドを実行します。

```bash
pip install -r requirements.txt
```

---

## ⚙️ 3. 初期設定 (settings.env)

投稿に必要な各設定を行います。

1. フォルダ内にある `settings.env.example` をコピーし、ファイル名を `settings.env` に変更します。
2. 作成した `settings.env` をメモ帳などのテキストエディタで開き、以下の必須項目を入力します。

| 設定項目 | 説明 | 記入例 |
| :--- | :--- | :--- |
| `YOUTUBE_CHANNEL_ID` | 監視したい YouTube チャンネル ID | `UCxxxxxxxxxxxxxxxxxxxxxxxx` |
| `BLUESKY_USERNAME` | Bluesky のハンドル名（@なし） | `yourname.bsky.social` |
| `BLUESKY_PASSWORD` | Bluesky のアプリパスワード | `xxxx-xxxx-xxxx-xxxx` |
| `POLL_INTERVAL_MINUTES` | 更新をチェックする間隔（分、最小5）| `15` |
| `APP_MODE` | 動作モード（まずは試しに自己確認用で設定） | `selfpost` |

> [!TIP]
> アプリ起動後は、画面（GUI）の **「⚙️ アプリ設定」** からでも、これらの項目を簡単に変更できます。

---

## 🚀 4. アプリの起動と確認

設定が終わったら、アプリを起動してみましょう。仮想環境が有効な状態で以下のコマンドを実行します。

```bash
python main_v3.py
```

### 初回起動時の注意点
一番最初に起動したとき、アプリは保存用のデータベースやフォルダを自動的に準備し、**一度自動で終了します**。  
これはツールが安全に初期化を行うための正常な動作です。

終了を確認したら、**もう一度同じコマンドを実行してください**。

```bash
python main_v3.py
```

専用の画面（GUI）が表示され、YouTube のチェックが開始されれば起動成功です！

---

## ✅ 5. 起動後のテスト

いきなり本番の投稿をする前に、正しく動くかテストしてみましょう。

1. **取得の確認**:  
アプリ画面上のリストに、設定したチャンネルの動画が表示されるか確認します。  
（空の場合は、少し待つかツールバーの「更新」ボタンを押してください）
2. **投稿テスト**:  
リストから動画をどれか1つ選び、上部の **「🧪 投稿テスト」** ボタンを押します。  
これはシミュレーションなので、**実際の Bluesky には何も投稿されません**。
3. **本番への移行**:  
準備が整ったら、「動作モード」を `autopost` (自動投稿) に変更するか、  
画面から手動で自由に選んで投稿を行ってください。

## 🆘 困ったときは（トラブルシューティング）

- **画面が開かない**:  
Python のバージョンが古いか、パッケージのインストール（ステップ 3）が失敗している可能性があります。
- **起動直後にすぐ閉じてしまう(初回起動時)**:  
初回起動時の仕様です。もう一度起動を試してください。
- **リストに動画が出ない**:  
YouTube のチャンネル ID が間違っている、またはネットワークの問題が考えられます。  
- チャンネルIDは、YouTubeのチャンネルページのURLに含まれる `UC` から始まる24文字の文字列です。  
`UC` から始まらない `https://www.youtube.com/@` から始まる文字列を使うには、  
`settings.env` の `YOUTUBE_API_KEY` にYouTubeDataAPIのキーを設定してください。

- **投稿に失敗する**:  
Bluesky のパスワードがログイン用になっていませんか？必ず「アプリパスワード」を設定してください。

- **その他のエラー**:  
すべてのエラーは `logs/app.log` というファイルに記録されます。  
原因が分からない場合はこちらのファイルを確認してください。
