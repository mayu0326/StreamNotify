# 🧪 投稿テストとデバッグ

StreamNotify には、実際に Bluesky などの SNS へ投稿する前に、  
意図した通りの文章や画像になっているかを確認するための「**投稿テスト（ドライラン）機能**」が備わっています。

また、アプリのより詳しい裏側の動きを確認するための「**デバッグモード**」も用意されています。

---

## 1. 🧪 投稿テスト（ドライラン）機能の使い方

実際に SNS へ投稿することなく、アプリ内で投稿までの流れをシミュレーションします。

### 💡 投稿テストのメリット
- **SNS のタイムラインを汚さない**:  
ミスがあってもフォロワーには見えません。
- **何度でもやり直せる**:  
実際の投稿フラグが立たないため、納得いくまでテンプレートなどを調整できます。
- **画像のリサイズ確認**:  
実際の投稿前に、画像が正しく処理（1MB以下に圧縮など）されるか確認できます。

### 🚀 実行手順
1. アプリのメイン画面（動画一覧）で、テストしたい動画の左端のチェックボックス（☑️）を入れます。
2. 画面上部のツールバーにある **「🧪 投稿テスト」** ボタンをクリックします。
3. ポップアップ画面が開くので、そのまま **「🧪 投稿テストを実行」** などのボタンをクリックします。
4. 画面下の**ログ（黒い画面）**に結果が表示されます。
   - `✅ 投稿テスト成功（本投稿なし）` と出れば、実際の投稿時も成功する可能性が高いです！

## 2. 🐛 デバッグモードの設定

もし「エラーが出てテストがうまくいかない」「テンプレートの文字がうまく置き換わらない」  
といったトラブルが起きた場合は、**デバッグモード**を使って原因を調べることができます。

### 🔧 デバッグモードをオンにする手順
1. ツールバーの **「⚙️ 全体設定」** を開きます。
2. 「基本設定」のタブを開きます。
3. **「DEBUG（デバッグモード）」** のチェックを入れます（または `true` にします）。
4. 設定を保存して、アプリを一度再起動します。

### 📝 ログの確認方法
デバッグモードをオンにした状態で再度「投稿テスト」などを行うと、  
通常よりはるかに多くの情報（裏側の通信内容や画像のサイズ変化など）がログに表示されるようになります。

ログはアプリ内の画面のほか、アプリのフォルダに入っている **`logs/app.log`** というファイルにも保存されています。  
エラーが起きたときはこのファイルの中身を見ると、  
何が原因で失敗したのか（例：パスワード間違い、通信エラーなど）が詳しく記載されています。

> [!TIP]
> **普段は「デバッグモード」はオフにしておこう！**  
> デバッグモードは情報量が多すぎるため、ずっとオンにしておくと少しアプリの動作が重くなったり、  
ログファイルが巨大になったりすることがあります。  
原因がわかって直ったら、チェックを外してオフに戻すことをおすすめします。

## 3. 🛠️ よくあるテスト時のトラブル

- **Q: 誤って本当に投稿してしまった！**
  **A**: ボタンの押し間違いに注意してください。  
  「🧪 投稿テスト」ではなく、普通の「📤 投稿」ボタンを押すと実際に SNS へ送られてしまいます。

- **Q: ログに何も出ない……**
  **A**: デバッグモードの設定が反映されていない可能性があります。  
  一度アプリを終了し、立ち上げ直してから再度テストを行ってみてください。

- **Q: 「認証情報が不正です」と出る**
  **A**: Bluesky のユーザー名（@なし）やアプリパスワードの設定が  
  間違っている可能性が高いです。「⚙️ 設定」を見直してください。
