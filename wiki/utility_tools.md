# 🛠️ いざという時の便利ツール（ユーティリティ）

StreamNotify では、通常のアプリ画面からは行えない「データベースの直接修正」や、  
「キャッシュのリセット」などのために、**おまけのスクリプト（便利ツール）**を同梱しています。

「**テスト投稿しすぎて過去の動画のフラグがおかしくなった**」  
「**どうしても古い動画をもう一度投稿し直したい**」といったトラブル時にご活用ください。

> [!CAUTION]
> **必ずアプリ本体を閉じてから実行してください！**  
> アプリを起動したままこれらのツールを使用すると、**データベースファイルが壊れてしまう危険性**があります。

---

## ツールの場所
すべてのツールは `v3/utils/` フォルダの中に整理されて入っています。  
実行する際は、コマンドプロンプトやターミナル（黒い画面）から `python ファイル名.py` のように実行します。

---

## 1. 投稿フラグをリセットしたい（未投稿に戻したい）
**場所**: `v3/utils/database/reset_post_flag.py`

- もうすでに（またはテストで）Bluesky に投稿してしまったけど、  
「もう一度未投稿の状態に戻して投稿し直したい」という場合に使います。  
- 動画をデータベースから消さずにフラグだけを「未」に戻せます。

### **使い方**: 
  ```bash
  python v3/utils/database/reset_post_flag.py
  ```
- 実行すると、「どの動画のフラグをリセットしますか？ 動画のIDを入力してください」と聞かれるので、  
YouTube の動画IDを入力します。

---

## 2. キャッシュ情報を強制的に再構築したい
**場所**: `v3/utils/cache/build_video_cache.py`

StreamNotify は、無駄な通信を減らすために動画の情報を「キャッシュ」として保存しています。  
もし何らかの理由でこのキャッシュが壊れてしまったり、エラーが頻発するようになった場合に、  
全動画の情報を一から取得し直して修復します。

### **使い方**: 
  ```bash
  python v3/utils/cache/build_video_cache.py
  ```
> [!WARNING]  
> 大量の YouTube API を消費するため、**本当に困った時だけ**使用してください。  
> v3.2.0のアプリ画面にある全般設定の「`Live設定`」タブ→「`キャッシュ管理`」内にある  
`キャッシュ強制更新(全件)`が機能しないほど**深刻な不具合が起きた時にのみ**使用してください。

---

## 3. その他の診断ツール
以下のツールは、システムの中身を「見るだけ」の機能で、変更は加えないため安全に実行できます。

- **`check_db_state.py`**: 現在のデータベースに動画が何件あるか、異常なデータがないかをサッと検査して表示します。
- **`check_cache_file.py`**: キャッシュのファイルサイズや中身を確認します。

もしアプリの挙動がおかしくてサポートを求める場合などは、  
これらのツールの出力結果を添えてからご報告いただけると、原因究明が早くなります。
