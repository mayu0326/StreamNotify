# v4 クイックスタート（はじめての準備）

StreamNotify **v4** を使い始めるための準備手順を、はじめての方にも分かりやすく解説します。  
v3 向けの手順は [クイックスタート（v3）](getting_started) を参照してください。

> [!NOTE]
> バージョン`v4.0.x`では、Bluesky へのログインは **アプリパスワード**（専用のパスワード）での利用を前提としています。  
> 取得方法は [Bluesky アプリパスワード](bluesky_appw) を参照してください。

> [!TIP]
> 画面の専門用語や「なぜそう動くか」を短く知りたい場合は、[v4 概要](v4_overview) や [v4 のフォルダ構成](v4_folder_structure) もあわせて読むと安心です。

---

## 💻 1. 動作環境の準備

v4 を動かすために必要なものは次のとおりです。

- **OS**: Windows 10/11 を想定した説明が多いですが、macOS / Linux でも動作します。
- **Python**: 3.10 以上（できれば 3.12 以上を推奨）
- **インターネット**: 常時接続されていること
- **あらかじめ用意しておく情報**:
  - 監視したい **YouTube チャンネル ID**（`UC` から始まる 24 文字）  
    → 調べ方は [YouTube チャンネルID](youtube_id)
  - **Bluesky** のハンドル名（`@` なし）と **アプリパスワード**  
    → [Bluesky アプリパスワード](bluesky_appw)
  - （任意）ニコニコ動画を監視する場合は **ユーザー ID**  
    → [ニコニコ動画 連携設定ガイド](niconico_setup)
  - （任意）**Twitch** の配信を扱う場合は **v4** でセンター+Auth 連携  
    → [Twitch 連携設定ガイド（v4）](twitch_setup)

### YouTube の「新着の知らせ方」を選ぶ

v4 では、YouTube の新着をどう拾うかを設定で選べます。

| 設定の値（画面・設定ファイル上の名前） | ひとことで |
| :--- | :--- |
| **poll** | **定期的にチェックする方式** … あなたの PC が、決まった間隔で YouTube 側を確認します。<br>通知用のサーバーは不要で、設定が簡単です。 |
| **websub** | **通知が早く届く連携**（設定名では WebSub） … <br>作者が用意した **通知の受け渡しサーバー（いわゆるセンター）** 経由で、新着を素早く取り込めます。<br>利用するには **専用ID と認証キー** が必要です。 |

> [!TIP]
> 「早く届く連携」を使う手順や、受け取る情報については [WebSub（リアルタイム通知）の利用手順](websub_guide) を参照してください。  
> まずは難しく感じたら **`poll` から始める** のがおすすめです。

## 🛠️ 2. アプリ本体の入手とフォルダ

- **リリース版をダウンロードする**  
  GitHub の Releases などから、配布されているアーカイブを取得し、解凍します。
- **Git で取得する**  
  リポジトリを clone するか ZIP をダウンロードして解凍し、中の **`v4` フォルダ** を開けるようにします。

以降、コマンドを実行するときは **リポジトリのルート**（`v4` と同じ階層にあるフォルダ）を基準にするか、<br>`v4` フォルダに移動してから行うか、説明どおりに選んでください。

### Windows で迷わない起動（おすすめ）

解凍した**StreamNotifyフォルダ**に **ランチャー** があります。

- **`start_launcher.bat`** をダブルクリックすると、アプリの起動・セットアップ・修復ができます。
- 初めてのときは **「セットアップ / 修復」** をクリックすると`仮想環境の作成`や`起動準備`を**自動**で行います。
- ２回目からは **「起動」** をクリックすると起動します。
- `Git`でファイルを取得した場合、v1~vの4全てのバージョンが選択できます。
- `リリース`から特定バージョンをダウンロードした場合は、**ダウンロードしたバージョンのみ**が表示されます。
- 既存ユーザー(v1~v3のユーザー)もこのランチャーをご利用いただけます。

> [!TIP]
> ランチャーはルートの Python で動きます。v4 専用の `.venv` は **`v4` フォルダ内** に作る運用と相性がよいです。

## 🛠️ 3. 仮想環境とパッケージ（コマンドで行う場合）

他の Python ソフトと混ざらないよう、**仮想環境** の利用を推奨します。

**例: `v4` フォルダ内に仮想環境を作る（Windows）**

```bash
cd v4
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

macOS / Linux の場合は、有効化を `source .venv/bin/activate` に読み替えてください。

## ⚙️ 4. 初期設定（settings.env）
> [!TIP]
> *ランチャーからセットアップ*をした場合は、**この項目をスキップしてください**。

1. **`v4` フォルダ** に `settings.env` を作ります。  
   初回起動前など、ファイルが無い場合は、`settings.env.example` をコピーして `settings.env` を作ってください。

> [!IMPORTANT]
> `settings.env` にはパスワードやキーが入ります。<br>**他人に見せたり、インターネットに公開したりしないでください。**

> [!TIP]
> 起動後はアプリ画面の **「⚙️ 設定」** から、設定をを変更できます。<br>
> アカウント設定やWebSub設定などは、アプリ起動後にアプリの設定から行えます。

## 🚀 5. アプリの起動

### Windows で迷わない起動（おすすめ）

セットアップ時に使った **ランチャー** からアプリを起動できます。

- **`start_launcher.bat`** をダブルクリックして、 **「起動」** をクリックすると起動します。

### 手動での起動方法

仮想環境を有効にしたうえで、次のいずれかで起動します。

- リポジトリの **ルート** から:  
  `python v4/main_v4.py`
- **`v4` フォルダ内** から:  
  `python main_v4.py`

### v3 との違い（初回起動）

v4 では、初回にテンプレートやフォルダを準備しますが、**v3 のように「一度勝手に終了して、もう一度起動し直す」必要はありません**。  
そのままメイン画面（GUI）が開けば成功です。

裏側では、通知を受け取るための小さな待ち受け（Webhook）なども動きますが、普段意識しなくて大丈夫です。

## ✅ 6. 起動後のアカウント設定とテスト

### まずは次の確認・設定をしてください。
- Blueskyのアカウント設定
- YouTubeやTwitch、ニコニコ動画など新着動画を取得するアカウント設定
- Websubセンターを利用する時はセンターの接続設定
- 各サービス用の投稿テンプレートの確認と必要に応じての編集

### いきなり本番投稿の前に、次を試してください。

1. **リストに動画が出るか**  
   - **`poll`** のとき: **「📡 新着取得 / RSS更新」** で手動取得できます。  
   - **`websub`** のとき: **「🔄 再読込」** で通知サーバー側との同期や表示の更新を行えます。  
   センターに繋がらない時(サーバーメンテ時・設定不備時)は自動的に **定期チェック方式(poll)へ切り替わる** ことがあります。<br>そのときは **`🔌 WebSubに再接続`** や RSS 系ボタンが表示されます。
2. **投稿のテスト**  
   リストで動画を 1 件選び、**「📤 投稿」** を押します。開いた画面に **「🧪 投稿テスト」** があるので、**実際には投稿せず** 内容を確認できます。
3. **本番投稿**  
   問題なければ同じ画面から本番投稿します。  
   動作モードを **`autopost`** にすると、手動ボタンが使えなくなり、自動で投稿する流れになります（[自動投稿ガイド](autopost_guide) も参照）。

## 🆘 困ったときは（トラブルシューティング）

- **画面が開かない**  
  Python のバージョン、`pip install` の成否を確認してください。
- **リストに動画が出ない**  
  チャンネル ID の誤り、インターネット接続の状態、`YOUTUBE_FEED_MODE` とセンター設定の組み合わせを確認してください。<br>`@` 形式のチャンネル名を使うには [YouTube API キー](youtube_api) が必要なことがあります。
- **通知の早い連携（websub）がうまくいかない**  
  [WebSub 利用手順](websub_guide) と、設定画面の接続テストを確認してください。一時的に **`poll`** にすれば、定期チェックだけで動かせます。
- **Bluesky に投稿できない**  
  ログイン用パスワードではなく **アプリパスワード** になっているか確認してください。
- **ログの場所**  
  くわしい記録は **`v4/logs/`** にあります。まず **`app.log`** と **`error.log`** を確認し、内容に応じて他のファイルも見るとよいです（ [v4 のログファイル案内](v4_logs_guide) ）。
