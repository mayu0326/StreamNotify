# StreamNotify v4 概要

このページでは、**v4** がどんなアプリか、v3 と並べて使う人向けに「違いが分かる程度」にやさしく説明します。  
一般的な StreamNotify の紹介は [StreamNotify 概要](overview) を、準備手順は [v4 クイックスタート](v4_getting_started) を先に読むとスムーズです。

---

## 🌟 StreamNotify v4 とは？

v4 も v3 と同じく、**あなたのパソコン上で動く**アプリです。  
YouTube やニコニコ動画などの新着やライブの情報を集め、**Bluesky** にお知らせ（ポスト）する手助けをします。

データベースや設定の多くは **PC の中（v4 用フォルダ）** に保存される考え方は v3 と同じです。

---

## ✨ v4 で意識するとよいこと（2〜3 点）

### 1. YouTube の新着の取り方を選べる

v4 では、次のどちらか（または状況に応じた切り替え）を前提にしています。

- **定期的に自分で確認する方式（poll）**  
  決まった間隔で YouTube 側をチェックします。通知用サーバーは不要で、入門に向いています。
- **通知が早く届く連携（設定名では WebSub）**  
  作者が用意した **通知の受け渡しサーバー** 経由で素早く情報を受け取れます。<br>利用には **運営から渡された ID・キー** が必要です。くわしくは [WebSub 利用手順](websub_guide) へ。

サーバーにつながらないとき(サーバーメンテ時や設定不備時など)は、自動的に **定期チェック方式(poll)へ切り替わる** ことがあります。<br>落ち着いたら、 **再接続** の操作で元に戻せます（ [v4 GUI 操作マニュアル](v4_gui_usermanual) 参照）。

### 2. 裏で「待ち受け」が動くことがある

通知の早い連携を使うとき、インターネット側からあなたの PC に通知を届けるため、**裏で小さな受け口（Webhook）** が動きます。  
普段は意識しなくて大丈夫ですが、セキュリティソフトやファイアウォールでブロックされていないか、うまく動かないときは確認の対象になります。

### 3. 画面のボタンが「モード」で少し変わる

上記の取り方や、通知サーバーとの接続状態によって、**ツールバーのボタン（新着取得・再接続など）の見え方** が変わります。  
くわしくは [v4 GUI 操作マニュアル](v4_gui_usermanual) で一覧しています。

### 4. Twitch 配信（オプション）

**Twitch** の配信開始・終了などを扱う場合は、**通知サーバー経由のモード（websub）** と **画面からの Twitch ログイン連携** が必要です。<br>手順は [Twitch 連携設定ガイド（v4）](twitch_setup) を参照してください。

## 🎛️ 動作モード

投稿を「自分で選ぶ」か「自動」かは **動作モード**（`APP_MODE` / 設定画面）で決まります。<br>名前は v3 に揃えていますが、**起動時の扱いは v3 と同じではありません**（下記注記）。

| モード | ざっくりした動き |
| :--- | :--- |
| **手動投稿（selfpost）** | 画面で動画を選んでから投稿します。最初のおすすめ。 |
| **自動投稿（autopost）** | 条件に合う新着を自動で投稿します。 |
| **テスト（dry_run）** | 実際には投稿せず、動作の確認に近い使い方ができます。 |
| **情報収集（collect）** | 設定で選んだときの区分（集めるだけに寄せた運用向け）。 |

> **v3 との違い:** v3 では初回起動時やデータベースファイル無い、または破損しているなどで使えないと判断された場合、<br>収集専用モードで起動し、**初回の取得のあとアプリが終了する**という仕様になっていました。<br>v4 にはその **自動移行はなく**、データベースに問題がある時は新規作成してそのまま使うという仕様に変更されました。<br>`v4/data/client_v4.db` は起動時に作成・初期化されます。

くわしい設定名は [設定項目の解説](config_reference) や [自動投稿ガイド](autopost_guide) を参照してください。<br>（v3 向けの説明が混じるページは、v4 では挙動が異なる箇所に注意してください）
