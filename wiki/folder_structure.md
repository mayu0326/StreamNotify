# 📁 ファイル・フォルダ構成について

StreamNotify があなたのパソコンの中でどのようにデータを保存しているのか、  
主要なフォルダやファイルの役割を解説します。
> この文書は現在の最新バージョンであるv3.2.0を基準としています。  
> そのため既存バージョンを利用中の場合、一部の機能や説明が異なる場合があります。

---

## 📂 StreamNotify の中身

アプリのフォルダを開くと、以下のような構成になっています。  
普段よく使うのは `data`, `images`, `templates` などのフォルダと、一番下にある `settings.env` です。

```text
StreamNotify/
├── v3/                   # アプリ本体が入っているフォルダ
│   ├── main_v3.py        # アプリを起動するファイル
│   ├── data/             # データベースの保存先 🟢
│   ├── images/           # サムネイル画像などの保存先 🖼️
│   ├── templates/        # 投稿メッセージのひな型の保存先 📝
│   ├── logs/             # 動作ログの保存先 📝
│   ├── plugins/          # 拡張機能 (ニコニコなど)
│   └── settings.env      # 設定ファイル ⚙️
```

### ⚙️ settings.env（設定ファイル）
- 最も重要なファイルです。あなたの Bluesky アプリパスワードや設定項目がすべてここに保存されています。
- アプリをアップデートしたり別の場所に移動する時は、  
**このファイルだけをコピーして持っていけば、これまでの設定をすべて引き継ぐことができます**。
- 設定をミスして壊れてしまった時のために、  
設定変更時にアプリが自動で `settings.env.backup_...` という**バックアップファイル**を作ります。

### 🟢 data/ （データベース）
- `video_list.db`や`youtube_video_detail_cache.json`、`deleted_videos.json` という  
巨大なデータベースファイルが入っています。
- ここには、今まで StreamNotify が見つけてきた「すべての動画のタイトルやURL」と、  
「Bluesky に投稿したかどうかの印（フラグ）」が記録されています。
- このファイルを消してしまうと、「過去に投稿した動画」をもう一度新着と勘違いして  
再投稿してしまうので注意してください。

### 🖼️ images/ （画像）
- 投稿する時に「1MB制限」に引っかからないように、アプリが自動で圧縮・リサイズした画像が  
一時的にここに保存されます。
- `default/`というフォルダと、その中の`noimage.png`という画像だけは消さないようにしてください。  
（サムネイルが取得できなかった時の代わりの画像です）。

### 📝 templates/ （テンプレート）
- `youtube` や `niconico` などのフォルダに分かれており、  
`yt_new_video_template.txt` などのテキストファイルが入っています。
- アプリの画面からだけでなく、メモ帳などでこのファイルを直接編集してもカスタマイズ可能です。
