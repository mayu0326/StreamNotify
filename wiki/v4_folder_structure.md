# v4 のフォルダ構成（ファイルの場所）

StreamNotify **v4** が、あなたのパソコンの中でどこに何を保存するかを **ざっくり** 説明します。  
v3 のフォルダ説明は [ファイル・フォルダ構成について](folder_structure) を参照してください（**v3 は `v3` フォルダ中心**、v4 は **`v4` フォルダ中心** で場所が異なります）。

---

## 📂 まず覚えておく場所（よく触る・困ったとき用）

| 場所 | 何のため？ |
| :--- | :--- |
| **`v4/settings.env`** | アカウントや動作の設定が入る **いちばん大事なテキストファイル** です。<br>バックアップを推奨します。 |
| **`v4/logs/*.log`** | 動作の記録（ログ）です。<br>不具合のときに開かれることが多いです。ファイルが複数あり、**役割ごとに分かれています**（ [v4 のログファイル案内](v4_logs_guide) ）。 |
| **`v4/data/client_v4.db`** | 動画一覧や投稿状態などを覚えておく **データベース** です。<br>基本的に`v4/data`にあるファイルを消すと問題がでますので、注意してください。 |

---

## 📁 `v4` フォルダの中身（目安）

```text
（リポジトリのルート）/
├── v4/
│   ├── main_v4.py          # 起動する Python ファイル
│   ├── settings.env        # 設定（自分用。他人に見せない）
│   ├── settings.env.example# 設定の例（ひな型）
│   ├── Asset/              # 配布用のひな型テンプレ・画像（下記）
│   ├── templates/          # 実際に使う投稿テンプレ（編集するならここ）
│   ├── images/             # サムネイルの取り込み・自動保存など
│   ├── data/               # データベース（client_v4.db など）
│   ├── logs/               # app.log など
│   └── backups/            # バックアップを保存するときの既定場所など
```

正確なファイル名はバージョンで増減することがあります。

---

## 🎨 `Asset` フォルダと `templates` / `images`

`v4/Asset/` には、**作者が同梱したひな型**（テンプレートやデフォルト画像）が入っています。

- **アプリ起動時** に、足りないファイルが **`v4/templates/`** などへ **自動でコピー** されることがあります。
- **すでにあるファイルは上書きしない** 設計なので、**自分で編集した内容は勝手に消えにくい** 考え方です。
- **文章や画像をカスタムしたい** ときは、基本 **`v4/templates/`** や **`v4/images/`** 側を編集します（ [テンプレートのカスタマイズ](template) ）。

> [!TIP]
> `Asset` 内の README に、フォルダの役割がさらに詳しく書かれています（開発者向けの表現も含みます）。  
> 一般利用では「触るのは主に `templates` と `images`」と覚えておけば十分なことが多いです。

---

## 🖼️ 画像まわりの注意

v4 では、サイト別の画像フォルダ（YouTube / Niconico / Twitch など）の下に、**取り込み用・自動保存用** のサブフォルダが作られることがあります。  
**`default` フォルダの画像**（noimage など）は、編集したり削除しないようにしてください。(起動に必要なファイルです)
