# ⚡ WebSub (リアルタイム通知) の利用手順

> [!WARNING]
> この機能は現在、**一部のサポーター様向けのベータ版（お試し機能）** となっております。  
管理者から専用の ID とキーの発行を受けた方のみが利用可能です。

---

## 1. WebSub（ウェブサブ）とは？

通常、StreamNotify は YouTube の「 **RSS** 」という仕組みを使って、  
新しい動画がないか「定期的に（例：15分ごとに）自分から見にいく」という方法（**ポーリング**）で動画を監視しています。

これに対し、**WebSub** という仕組みを使うと、  
動画が上がった瞬間に YouTube 側から「新しい動画が出たよ！」と **プッシュ通知（連絡）** を送ってもらえるようになります。

### 💡 メリット
- **超高速**:  
今まで最大15分ほどかかっていたお知らせが、動画公開後、数秒〜数分で届くようになります。
- **エコ**:  
無駄な通信が減り、YouTube の API 制限（クォータ）を節約でき、PC やサーバーへの負担も軽くなります。

---

## 2. センターサーバー方式について

普通、WebSub（プッシュ通知）を受け取るには、自分のパソコンのセキュリティ（ポート）を開けて、  
インターネットから直接アクセス可能にしないといけないため、**非常に危険で難しい設定**が必要です。

そこで StreamNotify v3 では、作者が管理する「 **センターサーバー**」を用意しています。  
YouTube からの通知は一旦この安全な **センターサーバーが代わりにすべて受け取り**、  
各ユーザーのアプリは**センターサーバー**に見に行く仕組みになっています。  
- センターサーバーを利用すれば、必要なものは**パソコンとネット回線とアプリだけ**になります。  
- YouTubeの監視設定を`ハンドル`ではなく`チャンネルID`で設定している場合、  
**YouTubeAPIキーの取得をしなくても**正確な情報（動画のURLやタイトルなど）を取得できる(※1)ので、  
**難しい設定や面倒な手続きは一切不要**です。

- ※1：YouTubeAPIキーをアプリに設定しない場合、センターサーバーのメンテナンス時や、  
センターサーバーが停止している場合は、正確な情報（動画のURLやタイトルなど）を取得できません。  

---

## 3. 利用手順・設定方法

管理者から以下の3つの情報を受け取っている必要があります。
- **WebSub クライアント ID (`WEBSUB_CLIENT_ID`)**
- **WebSub サーバーURL (`WEBSUB_SERVER_URL`)**
- **WebSub API キー (`WEBSUB_CLIENT_API_KEY`)**

### 画面（GUI）から設定する
1. ツールバーの **「⚙️ 全体設定」** を開きます。
2. 基本設定タブにある「**YouTube フィード取得モード**」を、通常の `poll` から `websub` に変更します。
3. アカウントタブの「WebSub」にある「**WebSub クライアント ID**」「**WebSubサーバーURL**」  
「**クライアント API キー**」に、それぞれ発行された `ID`、`URL`、`API キー` を入力します。
4. 設定を保存し、アプリを再起動します。

これだけで、リアルタイムで超高速なお知らせ機能が使えるようになります。

---

## 4. もしサーバーが落ちたら？

センターサーバーがメンテナンスなどで一時的にストップしていても安心してください。  
エラーが起きた場合は、自動的に通常の「定期確認（RSS・ポーリング）」モードに切り替わり  
（**フォールバック機能**）、見落としがないように設計されています。
