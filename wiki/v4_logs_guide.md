# v4 のログファイル案内

StreamNotify **v4** は、動作の記録を **`v4/logs/`** フォルダ内の **複数のテキストファイル** に分けて書き出します。  
トラブル時は **まず `app.log` と `error.log`** を開くとよいです。<br>設定で詳しさ（ログレベル）や保存日数を変えられます（ [設定項目の解説](config_reference) のログ関連）。

> [!TIP]
> ログは **1 日ごとにローテーション**（日付が変わると新しいファイルが作られ、古いものは名前に日付が付いて残る）します。<br>**既定では約 14 日分** を上限として残る設定になっています（`LOG_RETENTION_DAYS`）。

---

## 📁 置き場所

すべて **`v4/logs/`** にあります（アプリを別フォルダに移した場合は、その中の `logs` です）。

---

## 📋 ファイルの種類と「何が書かれるか」

次の表は **ふだんの調べ方** 向けの目安です。内部のロガー名までは細かく書いていません。

| ファイル名 | こんな内容が入ります |
| :--- | :--- |
| **app.log** | **全体の「ふつうの記録」**（起動、メイン処理、センター同期まわりの `v4.gui.adapter`、<br>マッピングされていない部品からの INFO など）。<br>**いちばん最初に見るファイル**のひとつです。 |
| **error.log** | **警告・エラー**（WARNING 以上）。不具合の手がかりを優先したいときに開きます。<br>`app.log` にも流れない「重いエラー」の集まりです。 |
| **gui.log** | **画面（GUI）本体**まわり（`v4.gui`）。<br>ウィンドウ操作や一覧表示など、画面側の処理の記録が中心です。 |
| **youtube.log** | **YouTube** の取得・RSS・ライブ判定、**センターとの WebSub クライアント**<br>（`v4.websub_client` など）に関する記録。 |
| **niconico.log** | **ニコニコ動画** の監視ワーカーやクライアントの記録。 |
| **thumbnails.log** | **サムネイル・画像**（ダウンロード、加工、`v4.images` など）と<br> **Asset（テンプレ配置）**（`v4.core.asset_manager`）の記録。 |
| **auth.log** | **認証まわり**（Twitch やその他 OAuth フローで使う `v4.core.auth`）。 |
| **webhook.log** | **ローカル待ち受け（Webhook）** でセンターなどから通知を受け取ったときの記録（`v4.webhook`）。 |
| **twitch.log** | **Twitch** のクライアント・イベント処理、および **EventSub の購読確保** に関する通信の一部<br>（`httpx` の特定リクエスト）。 |
| **post.log** | **Bluesky への投稿処理**（`PostLogger` / `v4.bluesky`）の **通常レベル**（成功・試行など）。 |
| **post_error.log** | Bluesky投稿のログのうち、**エラー寄り**に分けて残す記録（レベルは設定で変えられます）。 |

### 補足（内部の動き）

- **`v4.main`（起動処理）** や **`v4.setup`（初回セットアップ）**、表に名前がないログの多くは、**`app.log` / `error.log`** に載ります。
- **`v4.gui.adapter`**（一覧の再読込やセンター同期など）は、**`app.log` 側** に載ります。（画面専用の `gui.log` には出ません）
- **コンソール**（ターミナル）にも、設定したレベルに応じて同じような内容が流れます。

---

## 🔧 詳しさを変えたいとき

- 設定画面から**デバッグモード(`DEBUG_MODE=true`)** にすると、調べごと用の **より細かい内容** がログに出力されます。
- 機能ごとのログの細かさレベルは **`settings.env`** や **「⚙️ 設定」** から変更できます。<br>名前の対応は [設定項目の解説](config_reference) を参照してください。

---

## 🆘 困ったときの見る順（例）

1. **`v4/logs/error.log`** … いま起きている異常のメッセージがないか。  
2. **`v4/logs/app.log`** … 直前までアプリが何をしていたか。  
3. 症状に応じて **youtube.log** / **webhook.log** / **post_error.log** / **twitch.log** などを開く。
